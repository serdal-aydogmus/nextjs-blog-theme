## Mobile-First Approach

Start designing for the smallest screen first, then progressively enhance for larger devices:

```css
/* Mobile styles first */
.container {
  padding: 1rem;
}

/* Then tablet */
@media (min-width: 768px) {
  .container {
    padding: 2rem;
  }
}

/* Finally desktop */
@media (min-width: 1024px) {
  .container {
    padding: 3rem;
  }
}
```

## Flexible Grid Systems

Use relative units instead of fixed pixels:

- **Percentages** for widths
- **Em/rem** for typography
- **Viewport units** for full-screen elements

## Optimized Images

Implement responsive images for better performance:

```html
<img 
  src="image-small.jpg"
  srcset="image-small.jpg 480w, 
          image-medium.jpg 768w, 
          image-large.jpg 1200w"
  sizes="(max-width: 480px) 100vw, 
         (max-width: 768px) 50vw, 
         33vw"
  alt="Responsive image example"
/>
```

## Touch-Friendly Design

Design for touch interactions:

- **Minimum 44px** touch targets
- **Adequate spacing** between clickable elements
- **Swipe gestures** for mobile navigation

## Performance Considerations

- **Minimize HTTP requests**
- **Optimize critical rendering path**
- **Use efficient CSS selectors**
- **Implement lazy loading**

## Testing Across Devices

Always test your responsive designs on:

- Real devices when possible
- Browser developer tools
- Online testing platforms
- Different orientations

Responsive design is about creating seamless experiences across all devices. These principles will help you build websites that truly work everywhere.
